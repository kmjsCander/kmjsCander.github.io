表单文档

目前主要是配合模块化的使用

JSON配置

在module组件中，params传入一下类型的JSON即可获得一个表单

表单动态入参数

通过useForm的params传入
可参看模块化中的配置

key 类型 说明
valid {[key:string]:{trigger: ‘blur’,validator: (rule: 当前的校验配置项, value: any, callback: (err?: Error) => void) => any}[]} 对某个值的自定义校验同自定义校验
inputParams {[key: string]: any;} 对某个表单项的自定义HTML属性或props的配置

表单对外事件

监听handler触发

key 类型 说明
submit 点击提交按钮触发，进行表单校验并返回表单数据 —-
reset 还原表单为初始状态 —-

关于自定义校验

参数validFlexible 接受 类型是一个数组

key 类型 说明 默认
valid 正则字符串 正则表达式 -
message string 错误提示信息 -
trigger string 触发的方法 change

表单对外接口

通过useForm返回的methods进行函数调用

接口KEY 类型 接口说明
getData () => Promise<{ [l: string]: any }> 触发校验并返回校验结果
setData ({[l:string]: any}) => void 设置表单的值
reset () => { [l: string]: any } 还原表单为初始状态,并返回表单的值
getNoVerifyData () => { [l: string]: any } 获取表单当前用户输入的值，但是没有通过校验

如何使用表单的slot

使用表单的slot组件，可以完成部分复杂或与业务具有强耦合的表单控件。可以实现整个表单的一致性，以及全表单提交时，触发校验

具体的demo可以在项目中src/pages/demo/components/formSlotDemo.tsx查看demo
